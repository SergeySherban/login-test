Задание 1

Ответы:
    1. Пароли обязательно нужно хешировать, ни в коем случае не хранить в исходном виде. Это обусловлено тем что если Базу Данных будет взломано, то доступы к аккаунтам пользователей будут все равно не доступны. Хеш-функции предназначены для одностороннего преобразования данных, что делает невозможным восстановление исходного пароля из его хеша.

    2. Насколько данная система пригодна для аутентификации с использованием социальных сетей?
       Для реализации аутентификации с использованием социальных сетей лучше использовать специализированные инструменты и библиотеки, которые поддерживают соответствующие протоколы аутентификации, такие как OAuth или OpenID Connect. Это позволит обеспечить безопасность и удобство для пользователей при входе на вашем сайте через социальные сети.

    3. Перечислите функциональные требования для добавления к системе возможности регистрации.
        1. Форма регистрации с нужными полями для будущего профиля
        2. Проверка уникальности электронной почты
        3. Хеширование пароля
        4. Подтверждение пароля: чтобы пользователи не допускали опечаток при его вводе.
        5. Валидация данных
        6. Запоминание данных пользователя
        7. Уведомления об успешной регистрации: отображение сообщения об успешной регистрации.
        Опционально: подтверждение электронной почты, капча и защита от спама.

    4. Какие еще функции можно добавить к системе?
        1. Восстановление пароля
        2. Управление профилем пользователя
        3. Социальная авторизация
        4. Пользовательские роли и права
        5. Уведомления и оповещения в баре профиля
